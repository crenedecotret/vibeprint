# Plan: Add Hamburger Menu with About and Preferences

## Overview
Add a hamburger (☰) button to the UI that opens a dropdown menu with:
- **"About VibePrint Studio…"** — opens an About modal
- **"Preferences…"** — placeholder for future functionality (disabled/greyed out)

## Implementation Details

### 1. Add state for About modal (`types.rs`)
- Add `show_about: bool` to `AppState` struct
- Initialize `show_about: false` in `AppState::new()`

### 2. Create About modal (`modals.rs`)
- Add `show_about(&mut self, ctx: &Context)` method to `impl App`
- Design a simple modal window with:
  - Application name: "VibePrint Studio"
  - Version: extract from `Cargo.toml` (or hardcode "0.1.0")
  - Description: "ICC-aware print layout engine built entirely in Rust"
  - Copyright / license notice (from LICENSE file or simple "MIT/Apache 2.0")
  - Optional: "Built with egui/eframe" credit
- Use `egui::Window` centered on screen with OK button to close

### 3. Wire modal in main loop (`main.rs`)
- In `update()` method, add:
  ```rust
  if self.state.show_about {
      self.show_about(ctx);
  }
  ```
  (Right before the other modal checks or after is fine)

### 4. Add hamburger button to left panel (`ui/left_panel.rs`)
- In `draw_left()`, modify the top toolbar (lines 18-44) to include a hamburger button
- Place at the start of the horizontal layout, before Back/Forward/Home
- Button label: "☰" (or "≡") with appropriate size
- On click, show a `egui::Popup` or use `ui.menu_button()` with dropdown:
  - Menu item: "About VibePrint Studio…" → sets `self.state.show_about = true`
  - Menu separator
  - Menu item: "Preferences…" → disabled (`ui.add_enabled(false, ...)` or similar)
- Ensure the popup persists across frames (store open state if needed, but menu_button handles this)

### 5. Styling & UX
- Use consistent button sizing with existing toolbar buttons (btn_size Vec2::new(24.0, 22.0))
- Ensure the dropdown menu looks native/egui-idiomatic
- The About modal should be non-resizable, centered, and have a single Close/OK button

## Files to modify
- `src/bin/studio/types.rs` (state flag)
- `src/bin/studio/ui/modals.rs` (About modal implementation)
- `src/bin/studio/main.rs` (call show_about in update)
- `src/bin/studio/ui/left_panel.rs` (hamburger button + dropdown menu)

## Estimated complexity: Low
All changes are additive within existing patterns. No external dependencies or breaking changes.
